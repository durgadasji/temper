Static baseline (theme.css)

Component preview

One semantic token schema, five appearance modes on a single luminance ladder. This page shows two layers. The static baseline (top right) applies the five hand-tuned palettes from theme.css with no solving. The live tuner (bottom right) derives a full palette on the fly from three primitives: a tone notch, a color family, and a vision setting. The tuner also carries a Simulate control that previews a color-vision deficiency. Both layers persist in this browser; the page loads with the tuner on System.

Status indicators

An icon-only status dot is the case where color could become the sole carrier of meaning. Each dot carries a data-letter. Set the tuner's Vision control to anything other than Default and a letter appears beside every dot, and links gain underlines, straight from theme.css.

Callouts

Note

When your system prefers dark, System resolves to the deep Dark palette. Prefer the gentler Soft Dark as the everyday dark environment? The tuner and theme.css both document how to make that the System-dark target.

Passed

All body-text pairs meet WCAG AA in every mode.

Heads up

Keep accent chroma restrained if you fork the palette; loud accents break the calm.

Error

Danger foreground clears 4.5:1 on base, so error text stays legible.

Code

Inline tokens such as --color-surface read from the active mode. Block code uses code/background: 

/* apply a mode */
document.documentElement.dataset.theme = "soft-dark";

/* read a token in JS */
getComputedStyle(document.documentElement)
  .getPropertyValue("--color-accent");
